Kết quả xác minh thực tế — May 2026
CONFIRMED R-001 · proxy.ts là thật và BẮT BUỘC trong Next.js 16
Next.js 16 (ra mắt Oct 2025) chính thức deprecate middleware.ts và đổi sang proxy.ts. Đây không phải report giả lập — là breaking change thật. middleware.ts vẫn hoạt động cho Edge Runtime nhưng deprecated và sẽ bị xóa. Next.js cung cấp codemod tự động migrate. Plan hiện tại "chờ confirm" là sai — nên rename ngay.
Action required nextjs.org/blog/next-16
CRITICAL CVE R-002 · CVE-2025-29927 — auth bypass qua middleware.ts
Đây là lý do thực sự cần migrate. CVE-2025-29927 (CVSS 9.1) cho phép attacker bypass toàn bộ auth trong middleware.ts bằng cách forge header x-middleware-subrequest. Ảnh hưởng Next.js <15.2.3 và <14.2.25. Plan hiện tại không đề cập CVE này — rủi ro rất cao nếu auth đang đặt ở middleware.
CVE CVSS 9.1 offsec.com / zscaler.com
Không chỉ rename file — phải chuyển auth logic ra khỏi proxy.ts, đặt vào Layout hoặc Route Handler. proxy.ts chỉ dùng cho routing/redirect, không dùng cho authentication.
CONFIRMED R-003 · YouTube API: UC→UU trick đã lỗi thời — contentDetails là đúng
Plan đề xuất bỏ regex UC→UU, dùng channels?part=contentDetails để lấy relatedPlaylists.uploads. Đây là cách chính thức được Google Docs khuyến nghị. HC... channel IDs (Handle-based) cũng được API trả về đúng qua contentDetails. Hướng đi đúng.
Plan đúng developers.google.com/youtube/v3
GAP FOUND R-004 · pg_cron: giới hạn 32 concurrent jobs trên Supabase
Supabase pg_cron v1.6.4 hỗ trợ tối đa 32 concurrent jobs, mỗi job dùng 1 DB connection. Plan thêm 2 cron jobs mới nhưng không đề cập kiểm tra connection pool. Nếu project đã có nhiều crons, có thể gây connection exhaustion. Cần space out timing và kiểm tra Observability Dashboard.
Cần bổ sung supabase.com/docs/guides/troubleshooting/pgcron
GAP FOUND R-005 · Turso cleanup trong Edge Function: thiếu error isolation
Plan đặt Turso cleanup vào đầu daily-scan / hourly-tracker. Nếu Turso cleanup fail (timeout, network), nó có thể block toàn bộ Edge Function. Không có error boundary rõ ràng trong kế hoạch.
Cần bổ sung try/catch
Imaginative Tester — Scenarios không được cover trong plan
Critical · Phase 4 T-001 · Auth bypass vẫn tồn tại sau khi rename proxy.ts
Scenario: Plan rename middleware.ts → proxy.ts nhưng giữ nguyên auth logic bên trong. Attacker gửi request với header x-middleware-subrequest: src/proxy → bypass toàn bộ authentication check.
Expected: 401 redirect. Actual: truy cập thẳng protected route. Plan cần explicitly move auth sang Layout server component hoặc Route Handler — không chỉ rename file.
Critical · Phase 5 T-002 · Race condition trong save_user_secret vẫn có thể xảy ra
Plan dùng ON CONFLICT (name) DO UPDATE để fix F-009. Nhưng nếu name column không có UNIQUE constraint, ON CONFLICT sẽ không trigger. Không thấy plan bổ sung constraint này trong migration.
Cần thêm ALTER TABLE vault ADD CONSTRAINT unique_secret_name UNIQUE (name); vào migration, trước ON CONFLICT clause.
High · Phase 6 T-003 · baseline_vph = NULL vs = 0 — guard clause có miss case
Plan thêm guard baseline_vph = 0 để skip video mới. Nhưng video mới upload có thể có baseline_vph = NULL thay vì 0, tùy logic insert. Guard chỉ check = 0 sẽ miss case NULL → vẫn bị blacklist nhầm.
Guard nên là: IF baseline_vph IS NULL OR baseline_vph = 0 THEN SKIP
High · Phase 4 T-004 · channelId regex: custom handles @username không pass
Plan dùng regex ^UC[a-zA-Z0-9_-]{22}$ để validate channelId. Nhưng YouTube hiện hỗ trợ custom handles (@username) và các format ID khác (HC...). User paste handle thay vì UC ID sẽ bị reject với lỗi không rõ ràng.
Cần xử lý: (1) detect nếu input là handle → resolve qua API trước khi validate, hoặc (2) hiển thị error message hướng dẫn dùng channel ID dạng UC...
High · Phase 3 T-005 · daily_blacklist cleanup: timezone mismatch
pg_cron chạy ở UTC. Cron job xóa blacklist "cũ hơn ngày hiện tại" có thể xóa nhầm record của ngày hôm nay nếu timezone của app khác UTC (F-011 đang fix timezone default nhưng cleanup cron chưa tính).
Cleanup query cần explicit: WHERE date < CURRENT_DATE AT TIME ZONE 'UTC' thay vì CURRENT_DATE mặc định.
Medium · Phase 2 T-006 · Rate limiting: check "đã tồn tại" có thể bị race condition
Plan check channel ID tồn tại trước khi invoke Edge Function. Nếu 2 requests đến đồng thời, cả 2 đều pass check "chưa tồn tại" → duplicate invocations. Check-then-act không atomic.
Cần upsert với UNIQUE constraint trên channel_id ở DB level, không chỉ application-level check.
Medium · Phase 5 T-007 · (auth) route group: flash có thể vẫn xảy ra nếu dùng client redirect
Tách /login vào (auth) route group với empty layout fix được flash Sidebar. Nhưng nếu redirect tới login được thực hiện client-side (useRouter.push), Sidebar có thể render 1 frame trước khi redirect. Cần đảm bảo redirect xảy ra ở server (middleware/proxy hoặc layout server component).
